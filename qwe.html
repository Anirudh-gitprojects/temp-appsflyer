🚀 AppsFlyer S2S API v3

Server-to-Server Events API — Complete Integration Guide

⚡ REST API v3 S2S (Server-to-Server) React Native Compatible

📡 How It Works

🔗 The endpoint is:
POST https://api3.appsflyer.com/inappevent/{app_id}
🔑 Authentication: You'll need an S2S key to authenticate.

⚠️ The key requirement: The appsflyer_id is mandatory and is used to attribute events. It's generated when a user first installs the app, so you must capture it from the SDK and store it on your backend (mapped to your user/customer ID).

🎯 Backend-Owned Lifecycle Events (Single Source of Truth)

✅ CRITICAL ARCHITECTURE RULE: The backend owns the three lifecycle events below and is the single source of truth for their delivery. ⚠️ Do NOT emit these from the frontend (React Native).

The frontend only handles AppsFlyer SDK initialization, fires custom in-app events, sets customer_user_id via setCustomerUserId(user.id), and retrieves the AppsFlyer ID. Your backend will send these three milestone events to AppsFlyer S2S API.

🎉 activated

Trigger: First booking received by the user

📌 Implementation note: Trigger inside booking creation handler — check total_bookings == 1 before writing to outbox.

✨ trial_started

Trigger: Trial officially starts

⏱️ Send when user begins free trial (subscription trial period activated).

💎 subscribed

Trigger: First paid subscription transaction

💰 Emit after successful payment confirmation for the first subscription. 
// ============================================
// BACKEND LOGIC: Event emission (single source of truth)
// ============================================

// 1) activated event (inside booking creation handler)
async function handleBookingCreation(userId, bookingData) {
  // Fetch user's total bookings count
  const totalBookings = await db.query(
    'SELECT COUNT(*) FROM bookings WHERE user_id = $1', [userId]
  );
  
  // Trigger activated only if this is the FIRST booking
  if (totalBookings === 0) {  // before insert, total is 0
    await sendToAppsFlyerOutbox({
      userId: userId,
      eventName: 'activated',
      eventValue: { af_content_id: 'first_booking', booking_id: bookingData.id }
    });
  }
  
  // ... proceed with booking creation
}

// 2) trial_started event (when subscription trial begins)
async function handleTrialStart(userId, subscriptionDetails) {
  await sendToAppsFlyerOutbox({
    userId: userId,
    eventName: 'trial_started',
    eventValue: {
      af_content_id: subscriptionDetails.planId,
      af_currency: 'USD',
      trial_duration_days: subscriptionDetails.trialDays
    }
  });
}

// 3) subscribed event (first paid subscription transaction)
async function handleFirstPaidSubscription(userId, transactionData) {
  // Ensure this is the first paid subscription (not trial conversion duplicate)
  const existingPaid = await db.query(
    'SELECT id FROM paid_subscriptions WHERE user_id = $1 LIMIT 1', [userId]
  );
  if (existingPaid.rowCount === 0) {
    await sendToAppsFlyerOutbox({
      userId: userId,
      eventName: 'subscribed',
      eventValue: {
        af_revenue: transactionData.amount,
        af_currency: transactionData.currency,
        af_content_id: transactionData.planId,
        af_receipt_id: transactionData.receiptId
      }
    });
  }
}

📱 FRONTEND IMPLEMENTATION — What Your React Native App NEEDS

✅ Best Practice: Send AppsFlyer ID to Backend on Every Login

Yes — best practice is to send it on login to a backend endpoint.

🎯 Implementation Steps:

1. Create/Use endpoint like: POST /api/analytics/appsflyer/link

2. Call it after auth success with:
  • user_id (or infer from token server-side)
  • appsflyer_id
  • platform (ios/android)
  • app_version (optional)

3. Do this on login/signup (upsert) — update the mapping in your database 
// ============================================
// FRONTEND: React Native Implementation
// ============================================

import appsFlyer from 'react-native-appsflyer';
import { Platform } from 'react-native';

// Function to send AppsFlyer ID to your backend
const linkAppsFlyerIdToUser = async (userId, appsFlyerId) => {
  try {
    const response = await fetch('https://your-api.com/api/analytics/appsflyer/link', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${authToken}`,
      },
      body: JSON.stringify({
        user_id: userId,
        appsflyer_id: appsFlyerId,
        platform: Platform.OS,           // 'ios' or 'android'
        app_version: '1.2.3',         // optional but recommended
      }),
    });
    
    if (response.ok) {
      console.log('✅ AppsFlyer ID linked successfully');
    }
  } catch (error) {
    console.error('❌ Failed to link AppsFlyer ID:', error);
  }
};

// Call this AFTER successful login/registration
const onLoginSuccess = async (user) => {
  // 1. Get AppsFlyer ID
  appsFlyer.getAppsFlyerUID(async (err, uid) => {
    if (!err && uid) {
      // 2. Send to your backend
      await linkAppsFlyerIdToUser(user.id, uid);
      
      // 3. Set Customer User ID on SDK
      appsFlyer.setCustomerUserId(user.id, (res) => {
        console.log('✅ CUID set on AppsFlyer', res);
      });
    }
  });
};

🖥️ BACKEND — Mapping Endpoint & Outbox Processor

// ============================================
// BACKEND: Express.js - Store mapping + outbox pattern for reliability
// ============================================

const express = require('express');
const router = express.Router();

// POST /api/analytics/appsflyer/link - store appsflyer_id mapping
router.post('/api/analytics/appsflyer/link', async (req, res) => {
  const { user_id, appsflyer_id, platform, app_version } = req.body;
  await db.query(`
    INSERT INTO user_appsflyer_mapping (user_id, appsflyer_id, platform, app_version, last_seen)
    VALUES ($1, $2, $3, $4, NOW())
    ON CONFLICT (user_id) 
    DO UPDATE SET appsflyer_id = EXCLUDED.appsflyer_id, last_seen = NOW()
  `, [user_id, appsflyer_id, platform, app_version]);
  res.json({ success: true });
});

// Outbox processor: fetch pending lifecycle events and send to AppsFlyer
async function processAppsFlyerOutbox() {
  const pendingEvents = await db.query(`
    SELECT * FROM appsflyer_outbox 
    WHERE status = 'pending' ORDER BY created_at LIMIT 100
  `);
  
  for (const evt of pendingEvents.rows) {
    // Get appsflyer_id from mapping table
    const mapping = await db.query(
      'SELECT appsflyer_id, platform FROM user_appsflyer_mapping WHERE user_id = $1',
      [evt.user_id]
    );
    if (!mapping.rows[0]) continue;
    
    const appId = mapping.rows[0].platform === 'ios' ? 'id1234567890' : 'com.yourapp.android';
    const payload = {
      appsflyer_id: mapping.rows[0].appsflyer_id,
      customer_user_id: evt.user_id,
      eventName: evt.event_name,
      eventValue: evt.event_value,
      eventTime: new Date().toISOString().replace('T', ' ').split('.')[0],
    };
    
    const response = await fetch(`https://api3.appsflyer.com/inappevent/${appId}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', 'authentication': process.env.S2S_TOKEN },
      body: JSON.stringify(payload),
    });
    
    if (response.ok) {
      await db.query('UPDATE appsflyer_outbox SET status = $1, sent_at = NOW() WHERE id = $2', ['sent', evt.id]);
    }
  }
}
📊 Outbox Table Schema:

CREATE TABLE appsflyer_outbox (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(255) NOT NULL,
  event_name VARCHAR(100) NOT NULL,
  event_value JSONB,
  status VARCHAR(20) DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT NOW(),
  sent_at TIMESTAMP
);
